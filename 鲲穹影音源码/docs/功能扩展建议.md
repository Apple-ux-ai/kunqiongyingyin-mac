# 鲲穹影音 - 功能扩展建议

基于对项目源码、优化分析报告和现有能力的梳理，整理出**可新增或可增强的功能**，按优先级与实现成本分类，便于排期与选型。

---

## 一、项目现有能力概览

| 类别 | 已有功能 |
|------|----------|
| **播放** | 本地/URL、播放列表、续播（进度记忆）、播放模式（列表循环等）、倍速、快进快退 |
| **画面** | 亮度/对比度/饱和度/色相、比例、预设滤镜、全景、旋转/翻转、截图 |
| **音频** | 音量、声道、外挂音轨、内置多音轨切换、环绕/3D 效果、同步延迟 |
| **字幕** | 主/次字幕、外挂、样式、延迟 |
| **系统** | 关灯模式、全屏、DLNA 投屏（UI）、默认播放器、协议/文件关联、多语言、登录、更新、反馈 |
| **设置** | 解码器/分离器/渲染器、热键、截图路径与格式 |

---

## 二、建议新增功能（按优先级）

### 高价值、用户常需求

| 功能 | 说明 | 实现要点 | 优先级 |
|------|------|----------|--------|
| **画中画（PiP）** | 小窗播放，边看边做其他事 | 使用 `document.pictureInPictureEnabled` + `<video>.requestPictureInPicture()`，在控制栏或右键菜单增加「画中画」入口 | P1 |
| **A-B 段循环** | 指定区间重复播放（学外语、练舞常用） | 在 `timeupdate` 里判断 `currentTime` 是否超出 B 点，超出则 seek 到 A 点；需 UI：设 A 点、设 B 点、清除、开关 | P1 |
| **收藏 / 我的最爱** | 将当前或列表项加入收藏，便于快速打开 | 已有语音/菜单入口但提示「开发中」；用 localStorage 或与续播同结构的 JSON 存 `filePath/url + 标题`，侧边或独立面板展示列表并支持点击播放 | P1 |
| **最近播放列表** | 独立入口查看并打开最近播放过的文件 | 续播已有 `playback_resume` 按时间戳；可复用或单独维护「最近 N 条」列表，在欢迎页或「文件」菜单下增加「最近播放」子菜单并展示列表 | P1 |
| **播放列表导入/导出** | 保存列表为文件、从文件恢复列表 | 导出：将 `playlist` 序列化为 JSON/M3U 等；导入：选择文件解析后 `addFileToPlaylist` 或批量加入；菜单「文件」下增加「导入列表」「导出列表」 | P2 |

### 体验与健壮性

| 功能 | 说明 | 实现要点 | 优先级 |
|------|------|----------|--------|
| **记忆窗口位置与大小** | 关闭前保存 bounds，启动时恢复 | 主进程在 `window.close` 前 `getBounds()` 存配置/本地文件，`createWindow` 时读取并传入；preload 需暴露 `getWindowBounds`/`setWindowBounds` 或由主进程直接读配置 | P2 |
| **截图默认路径与 Electron 一致** | 首次截图时默认保存到「图片」目录 | 见《项目优化分析报告》§16：用 `window.electronAPI` 判断环境；preload 新增 `getPath('pictures')`，主进程 `app.getPath('pictures')` 通过 IPC 返回；截图路径初始化改为调用该 API | P0（修 bug） |
| **转码/加载失败时的明确提示** | 避免用户只看到黑屏或无声 | 在转码失败、解码失败、网络错误等分支里，除 logger 外调用现有 `showAlert`/消息通知，提示「该格式需要转码，转码失败：原因」或「加载失败，请检查网络/文件」 | P2 |
| **网络请求失败可重试** | 广告、更新、登录等失败时给「重试」入口 | 在请求 catch 里设置状态 + 在对应 UI（如更新提示、登录面板）显示「加载失败，点击重试」按钮，点击再次发起请求 | P3 |

### 安全与合规（与优化报告一致）

| 功能 | 说明 | 实现要点 | 优先级 |
|------|------|----------|--------|
| **渲染进程去掉 require('electron')** | 避免崩溃、统一走 preload | 截图、紧凑模式等改为 `window.electronAPI` + 新增 IPC（如 `getPath`、窗口 bounds）；删除 `toggleCompactMode` 内 require 或改为 IPC 实现 | P0 |
| **open-external 仅允许 http(s)** | 防止 javascript:/file: 等风险 | 主进程校验 URL 协议白名单（已修复则保持） | P0 |
| **SECRET_KEY 环境变量化** | 密钥不进仓库 | 构建/运行从 `process.env` 或构建时注入读取 | P1 |
| **广告/API 的 URL 校验与 innerHTML 安全** | 防 XSS | 对 `adv_url`/`target_url` 做格式与协议校验；图片用 `setAttribute('src', ...)`，链接仅用于受校验的 openExternal | P2 |

## 三、可选增强（按需排期）

| 功能 | 说明 | 实现要点 |
|------|------|----------|
| **播放统计** | 总观看时长、每文件播放次数等 | 本地存储或与账号关联；在设置/关于里展示「本机统计」 |
| **单条播放列表项右键菜单** | 复制路径、在文件夹中显示、从列表移除、加入收藏等 | 与现有「在文件夹中显示」统一，扩展右键项并调现有 API |
| **倍速预设快捷按钮** | 0.5x / 1x / 1.25x / 1.5x / 2x 等一键切换 | 在控制栏或右键菜单增加若干按钮，点击设置 `playbackRate` |
| **关灯模式覆盖多显示器** | 多屏时每块屏都遮罩 | 主进程遍历 `screen.getAllDisplays()`，为每个 display 创建/关闭遮罩窗口 |
| **字幕实时翻译/机翻** | 外挂字幕或当前句机翻显示 | 需对接翻译 API；可先做「复制当前字幕」再扩展 |
| **播放列表拖拽排序** | 列表内拖拽调整顺序 | 用 HTML5 拖拽 API 或引入轻量 sortable，更新 `playlist` 数组与 `currentIndex` |

---

## 四、与《项目优化分析报告》的对应关系

- **先做优化报告中的 P0**：截图环境判断、去掉渲染进程 `require('electron')`、open-external 校验（若未做）、默认播放器 appId（已修复）。
- **本建议中的 P0/P1 功能**：在修完上述问题后优先加入画中画、A-B 循环、收藏、最近播放、播放列表导入导出、窗口记忆、截图路径与失败提示等，会明显提升可用性和口碑。
- **安全与合规项**：SECRET_KEY、广告 URL 校验等与报告一致，按报告排期即可。

---

## 五、建议实现顺序（简要）

1. **立即**：截图保存用 `window.electronAPI` 判断 + preload 暴露 `getPath('pictures')`；渲染进程去掉所有 `require('electron')`。
2. **短期**：画中画、A-B 段循环、收藏、最近播放、播放列表导入/导出。
3. **中期**：窗口位置/大小记忆、转码与加载失败提示、网络重试、广告/API 安全加固。
4. **长期**：播放统计、多显示器关灯、倍速快捷、字幕相关增强等按需求选做。

若你希望先落地某一项（例如画中画或 A-B 循环），可以指定功能名，我可以按当前代码结构给出具体修改位置和示例代码。
